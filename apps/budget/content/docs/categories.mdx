title: Categories
description: Managing income and expense categories.
---


Categories help organize transactions into meaningful groups for tracking and analysis.

## Overview

Categories are:

- **Pre-defined** - Default set for common needs
- **Typed** - Either income or expense
- **Visual** - With icons and colors

## Category Properties

| Property | Type                  | Description       |
| -------- | --------------------- | ----------------- |
| `id`     | string                | Unique identifier |
| `name`   | string                | Display name      |
| `type`   | 'income' \| 'expense' | Category type     |
| `icon`   | string                | Lucide icon name  |
| `color`  | string                | Hex color code    |

## Data Model

```typescript
interface Category {
  id: string;
  name: string;
  type: "income" | "expense";
  icon: string;
  color: string;
}
```

## Using Categories

### In Transactions

```typescript
// Creating a transaction with category
const transaction = {
  type: "expense",
  amount: 5000,
  category: "groceries", // Category ID
  description: "Weekly groceries",
  date: "2024-12-03",
};
```

### In Budgets

```typescript
// Creating a budget for a category
const budget = {
  category: "groceries", // Category ID
  amount: 15000,
  period: "monthly",
};
```

## Category API

### List Categories

```bash
GET /api/categories
```

Response:

```json
{
  "categories": [
    {
      "id": "salary",
      "name": "Salary",
      "type": "income",
      "icon": "Briefcase",
      "color": "#10B981"
    },
    ...
  ]
}
```

### Filter by Type

```bash
GET /api/categories?type=expense
```

## Best Practices

1. **Keep it Simple** - Don't create too many categories
2. **Be Consistent** - Always use the same category for similar items
3. **Use "Other"** - For rare one-off expenses
4. **Review Periodically** - Check if categories match your needs
